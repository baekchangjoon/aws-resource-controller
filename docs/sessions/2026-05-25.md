## 5. 라이브 검증 (Playwright MCP, 2026-05-25)

GitHub Actions의 자동 E2E와는 별도로, 본 세션에서 **Playwright MCP**로 라이브 사이트를 직접 인터랙티브하게 검증.

### 단계
1. `browser_navigate` → https://app-dev.dev-temp-mail.com (Title: "TempSES — 일회용 이메일")
2. 자동 발급된 주소 캡처: `77929b72@dev-temp-mail.com` ([screenshot](../screenshots/01_initial.png))
3. `aws sesv2 send-email`로 외부에서 메일 발송 (XSS payload 포함):
   ```html
   <p>Sent via <strong>Playwright MCP</strong> interactive run.</p>
   <script>alert("xss")</script>
   <img src="https://tracker.example.com/pixel.png"/>
   ```
4. `browser_wait_for` → 5초 폴링이 인박스 갱신, 1분 내 메일 도착 ([screenshot](../screenshots/02_inbox.png))
5. `browser_click` → 메일 선택 → iframe 렌더 ([screenshot](../screenshots/03_message_view.png))
6. `browser_evaluate`로 iframe srcdoc 검사:
   - `sandbox=""` ✅
   - `referrerpolicy="no-referrer"` ✅
   - `<script>` 태그 제거됨 ✅
   - `alert("xss")` 텍스트 제거됨 ✅
   - `tracker.example.com` 흔적 제거됨 ✅
   - CSP `default-src 'none'; img-src data:` 메타 존재 ✅
   - 안전 콘텐츠 ("Playwright MCP") 보존 ✅
7. "새 주소 발급" 클릭 → 새 주소 `ae87d3ad@dev-temp-mail.com`, 인박스 클리어 ([screenshot](../screenshots/04_new_address.png))

### 결론
**모든 보안/UX 요구사항이 라이브 환경에서 충족.** 서버 sanitize(bleach) + 클라이언트 iframe sandbox + CSP의 다층 방어가 실제 작동.

### 9.2 OIDC 배포 롤 권한 축소 ([#20](https://github.com/baekchangjoon/aws-resource-controller/pull/20))

- `terraform/modules/github_oidc/main.tf`: AWS-managed `AdministratorAccess` attachment 제거 → customer-managed `tempses-dev-github-deploy` policy로 교체.
- **Allow list (이 스택이 실제로 만지는 서비스만)**: `acm`, `apigateway`, `budgets`, `cloudfront`, `cloudwatch`, `dynamodb`, `iam`, `lambda`, `logs`, `route53`, `s3`, `ses`, `sns`, `sqs` + `sts:GetCallerIdentity` + `tag:GetResources`. Organizations / billing / KMS / EC2 / RDS 등은 implicit deny.
- **명시 Deny**: `account:*`, `organizations:*`, `iam:Create/Delete/UpdateUser`, `iam:Create/DeleteAccessKey`, `iam:*LoginProfile`, `iam:*UserPolicy` — 워크플로가 탈취돼도 사용자/액세스키 생성, 조직 조작 불가.
- **2단계 검증**:
  1. PR 머지 직후 자동 CD ([Run 26389961140](https://github.com/baekchangjoon/aws-resource-controller/actions/runs/26389961140))가 admin attachment 제거 + 새 policy attach까지 정상 적용.
  2. 이어 `workflow_dispatch`로 한 번 더 CD ([Run 26390034718](https://github.com/baekchangjoon/aws-resource-controller/actions/runs/26390034718)) — **새 policy만으로** 모든 단계 통과.

### 9.3 SNS 알람 이메일 구독 — Gmail prefetch 함정 우회

- 증상: `terraform apply -replace`로 구독을 재생성한 뒤 사용자가 confirmation 메일을 클릭(16:44 KST). 1분 뒤(16:45) `AWS Notification - Unsubscribe Confirmation`이 도착하며 구독 자동 비활성화. `aws sns list-subscriptions-by-topic`에서 `SubscriptionArn=Deleted`로 재현됨.
- 원인: Gmail / G Suite / 일부 corporate mail gateway가 confirmation 메일 안의 모든 링크를 자동 prefetch — `SubscribeURL` 옆에 같이 있는 `UnsubscribeURL`까지 따라가버려, confirm 직후 즉시 구독 취소.
- 해법: 메일의 `SubscribeURL`을 **클릭 대신 복사**해서 받아 `aws sns confirm-subscription --token … --authenticate-on-unsubscribe true` 로 API confirm. 이 옵션은 unsubscribe도 AWS 인증을 요구하게 만들어 prefetch 무력화.
- 검증: `get-subscription-attributes` 결과 `PendingConfirmation=false`, `ConfirmationWasAuthenticated=true`. 테스트 publish 메일(MessageId `1f79a33b-be41-54f6-ae0f-c1b5b8e8869f`) 정상 수신.
- 운영 지식으로 memory에 저장 (`feedback_sns_email_confirm.md`) — 다음에 다른 SNS email 구독을 만들 때 이 절차를 그대로 적용할 수 있도록.

### 9.5 WAF + API throttling (D7 보류 해제)

- 새 모듈 [`terraform/modules/waf/`](../../terraform/modules/waf/) — CloudFront-scope WAFv2 web ACL (`us-east-1`).
  - Priority 1: AWS Managed `CommonRuleSet` (OWASP — SQLi/XSS/path traversal).
  - Priority 2: AWS Managed `KnownBadInputsRuleSet`.
  - Priority 3: `IPRateLimit` (rate-based, **2000 req/5min/source IP**, action=block).
  - 기본 동작 Allow, 모든 룰 CloudWatch metrics + sample 활성.
- [`frontend`](../../terraform/modules/frontend/) 모듈에 `web_acl_arn` 변수 추가, `aws_cloudfront_distribution.web.web_acl_id`로 연결. 현재 distribution `E36YDK2L5SPTL7`에 `arn:aws:wafv2:us-east-1:322242916220:global/webacl/tempses-dev-web/c7d60c03-…` 부착됨.
- [`api`](../../terraform/modules/api/) 모듈의 `$default` stage에 `throttling_rate_limit=50`, `throttling_burst_limit=100` 적용. HTTP API v2가 WAFv2 attach를 지원하지 않으므로 stage 전체 RPS 상한으로 보강.
- [`github_oidc`](../../terraform/modules/github_oidc/) allow list에 `wafv2:*` 추가.
- **CD 1차 실패** ([Run 26391441383](https://github.com/baekchangjoon/aws-resource-controller/actions/runs/26391441383)): IAM 정책 전파 race — 같은 apply 안에서 OIDC 롤에 `wafv2:*` 추가 후 즉시 `wafv2:CreateWebACL` 호출 → `AccessDeniedException`. STS assumed-role의 cached permission set이 옛 정책.
- **CD 2차 성공** ([Run 26391522904](https://github.com/baekchangjoon/aws-resource-controller/actions/runs/26391522904)): 새 STS 토큰 + propagated 정책으로 즉시 통과. 일반화된 운영 지식으로 memory에 저장 (`feedback_iam_propagation_in_apply.md`).
- 배포 후 검증:
  - `aws wafv2 list-web-acls --scope CLOUDFRONT` — `tempses-dev-web` 생성 확인.
  - `aws cloudfront get-distribution` — `WebACLId`가 WAF ARN.
  - `aws apigatewayv2 get-stage` — `ThrottlingRateLimit=50.0`, `ThrottlingBurstLimit=100`.
  - `curl https://app-dev.dev-temp-mail.com/` → **200**, `POST /addresses` → **201**.
